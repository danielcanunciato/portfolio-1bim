**Projeto A:** O projeto apresenta várias características privadas de um sistema, sendo elas:

- Armazenamento de Biometria (privado);

- Integração da Folha de Pagamento (privado);

- Conta de Cliente (privado);

- Se esses tipos de conteúdos forem vazados em algum lugar, vários dados seriam perdidos e causariam enormes riscos à empresa ou repositório.


Uma medida de segurança que pode ser sugerido à este projeto poderia ser colocado em um repositório.

**Projeto B:** O projeto já é considerado público por conta de seus arquivos onde:

- São arquivos úteis para todos os desenvolvedores (público);

- Não necessariamente precisar de dependências externas (público);

- A opção de fazer ou não um portfólio (público);


O risco de ter informações ou arquivos públicos varia pois muitos códigos que estiverem abertos podem conter algo sensível que nem todos os desenvolvedores podem ter acesso, também apresenta um fato de não possuir nenhum dependência para poder acessar esses arquivos; Uma medida simples de segurança e eficaz que pode ser aplicada à este projeto é sempre revisar o que está sendo postado nos repositórios, onde então os torna mais sensíveis.
